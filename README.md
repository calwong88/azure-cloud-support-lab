# Azure Cloud Support Lab

## 🚀 Overview

This repository documents a hands-on Azure Cloud Support Lab designed to simulate real-world enterprise cloud administration and troubleshooting scenarios.

The purpose of this project is to gain practical experience managing and supporting Azure environments aligned with AZ-104 objectives and common Tier 1 / Tier 2 Cloud Support Engineer responsibilities.

The lab focuses on identity management, role-based access control (RBAC), networking, virtual machines, storage, monitoring, and structured troubleshooting.

---

## 🎯 Objectives

- Implement identity and access management using Azure Entra ID
- Apply least-privilege RBAC across users, groups, and resource groups
- Deploy and manage Windows and Linux virtual machines
- Configure virtual networks, subnets, and Network Security Groups (NSGs)
- Secure and manage Azure Storage accounts
- Monitor resources using Azure Monitor and Activity Logs
- Diagnose and resolve simulated cloud support incidents

---

## 🏗️ Lab Architecture

This lab simulates a small enterprise Azure environment including:

- Azure Entra ID tenant with test users and security groups
- Resource groups for workload separation
- Virtual Network with segmented subnets
- Network Security Groups enforcing traffic control
- Windows and Linux virtual machines
- Storage account with blob containers
- Azure Monitor for logging and diagnostics
